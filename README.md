# Mitigating TCP Incast via GPU-Accelerated AQM

Compares three queue engine designs under TCP incast traffic generated by an ns3 simulator.

| Engine | Sort | Priority |
|---|---|---|
| CPU FIFO | none (drop-tail) | no |
| CPU PQ | std::sort introsort O(n log n) | yes |
| GPU PQ | Thrust radix sort O(d·n) | yes |

## How it works

1. **ns3 simulator** generates incast traffic (N senders firing simultaneously) and writes PCAP files to disk
2. **Packet Replay** reads the PCAPs, stamps DSCP into each IP header based on sender identity, and streams binary frames to the queue engine over a TCP socket
3. **Queue engine** reads DSCP from the IP header directly and makes admit/drop/evict decisions

## Scenarios

| Scenario | Senders | Buffer | Expected drop rate |
|---|---|---|---|
| low | 128 | 50 MB | 0% (baseline) |
| medium | 5,000 | 5 MB | ~50% |
| high | 10,000 | 16 MB | ~48% |
| test | 500 | 1 MB | ~49% |

## Run

```bash
# build all engines (from src/)
cd src && make build

# run a single comparison (from src/)
cd src
make compare COMPARE_SCENARIO=test
```
